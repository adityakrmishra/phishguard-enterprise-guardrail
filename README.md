# 🛡️ PhishGuard Enterprise Guardrail

> **ET AI Hackathon · Problem Statement 5 — Compliance Guardrails & Auditable AI Decisions**

PhishGuard is a production-ready, AI-powered phishing detection pipeline built for Indian fintech and banking enterprises. It combines ultra-fast semantic triage with a fine-tuned LLM classifier and an explainability agent that generates auditable compliance reports — all exposed via a REST API and a B2B dashboard.

---

## ✨ Key Features

| Capability | Detail |
|---|---|
| ⚡ **Fast-path triage** | FAISS nearest-neighbour search resolves safe/scam in < 5 ms |
| 🧠 **Heavy-brain fallback** | DistilRoBERTa + LoRA adapter resolves edge cases fast-path can't classify |
| 📋 **Audit log** | Groq LLM generates `fraud_intent` + `compliance_reasoning` for every flagged message |
| 🗂️ **Synthetic training data** | Red-team dataset generated by an LLM prompt targeting Indian fintech phishing patterns |
| 🖥️ **B2B dashboard** | Streamlit UI with live metrics, session history, and an Audit Log expander |

---

## 🏗️ Tech Stack

| Layer | Technology |
|---|---|
| API | FastAPI + Uvicorn |
| Frontend | Streamlit |
| Fast-path classifier | FAISS (`faiss-cpu`) + `sentence-transformers` (`all-MiniLM-L6-v2`) |
| Heavy-brain classifier | `distilroberta-base` + PEFT LoRA (`peft`, `transformers`) |
| Explainability agent | Groq API (OpenAI-compatible) via `openai` SDK |
| Data generation | Groq / OpenAI / Together AI |
| Config | `python-dotenv` |

## 🚀 Setup & Run

### 1. Clone & create virtual environment

```powershell
git clone https://github.com/<your-username>/phishguard-enterprise-guardrail.git
cd phishguard-enterprise-guardrail

python -m venv .venv
.venv\Scripts\Activate.ps1       # Windows
# source .venv/bin/activate       # macOS / Linux
```

### 2. Install dependencies

```powershell
pip install -r requirements.txt
```

### 3. Configure environment variables

Create a `.env` file in the project root:

```ini
# Groq (free tier — recommended)
OPENAI_API_KEY=gsk_xxxxxxxxxxxxxxxxxxxx
OPENAI_BASE_URL=https://api.groq.com/openai/v1
OPENAI_MODEL=mixtral-8x7b-32768

# Or use OpenAI / Together AI — just change the values above
```

### 4. Generate the synthetic training dataset

```powershell
python -m src.ml_pipeline.data_generator
# Output: data/synthetic/red_team_phishing.csv
```

### 5. Fine-tune DistilRoBERTa

```powershell
python -m src.ml_pipeline.train_lora
# Output: models/distilroberta-finetuned/
```

### 6. Run the API server (Terminal 1)

```powershell
uvicorn src.api.main:app --host 0.0.0.0 --port 8000 --reload
# Swagger docs: http://localhost:8000/docs
```

### 7. Run the dashboard (Terminal 2)

```powershell
streamlit run frontend/app.py
# Dashboard: http://localhost:8501
```

## 🔌 API Reference

### `POST /api/v1/analyze`

**Request:**
```json
{ "text": "URGENT: Your account is suspended. Verify at secure-hdfc.in/verify" }
```

**Response:**
```json
{
  "verdict": "KNOWN_SCAM",
  "distance": 0.3127,
  "matched_template": "URGENT: Your account has been suspended...",
  "fraud_intent": "Authority Impersonation",
  "compliance_reasoning": "This message impersonates an RBI directive to coerce credential sharing, violating NPCI AML guidelines."
}
```

**Verdict values:**

| Value | Meaning |
|---|---|
| `SAFE` | Semantically close to legitimate banking communication |
| `KNOWN_SCAM` | Matches a catalogued phishing pattern |
| `ANOMALY_NEEDS_LLM` | Unresolved edge case (classifier unavailable) |
